## 一、目标检测任务的基本内涵

目标检测的任务目标是：在输入图像中同时完成**目标类别识别**与**目标空间定位**。与图像分类仅判断图像中是否存在某类对象不同，目标检测要求系统进一步指出每个目标在图像中的具体位置，因此其输出通常包括三项内容：目标类别、目标边界框以及相应的置信度分数。

边界框通常采用矩形框形式表示，可以使用中心点与宽高 (x,y,w,h) 来编码，也可以使用左上角和右下角坐标来表示。无论采用何种具体参数化方式，本质上都是对目标空间范围的回归。因此，目标检测本质上是一个同时包含**分类任务**与**回归任务**的联合学习问题：一方面需要完成离散类别判定，另一方面需要完成连续空间参数预测。

从任务场景上看，目标检测既可以针对单一类别目标，也可以处理多类别、多实例场景。在较简单的单目标检测中，系统主要关注某一种特定类别，如人脸检测或行人检测；而在更一般的多目标检测任务中，系统需要在同一幅图像中找出所有感兴趣目标，并分别完成类别标注与位置估计。这一要求使得目标检测显著复杂于普通图像分类，因为它不仅要识别“是什么”，还必须回答“在哪里”“有多少个”“是否重叠”等问题。

此外，在检测框架中，“背景（background）”通常被显式考虑。由于图像中绝大多数区域并不对应任何有效目标，如果不对背景建模，模型容易产生大量误检。因此，无论是传统检测器还是深度学习检测器，都会在训练过程中引入前景—背景区分机制，以增强模型对非目标区域的抑制能力。

## 二、目标检测的评价指标与后处理机制

目标检测的性能评估不仅需要考察类别判定是否正确，还必须衡量预测框与真实框之间的位置重合程度。在这方面，最基本也是最重要的指标是 **IoU（Intersection over Union，交并比）**。IoU 定义为预测框与真实框交集面积占并集面积的比例，用于反映检测框的定位精度。IoU 值越高，说明预测框与真实框越重合。通常，在评估时会设置一个阈值，例如 IoU 大于 0.5 时，认为该检测结果是一次正确命中。

在整体检测效果方面，除了定位精度外，还需要考虑模型是否存在误检与漏检问题。因此，**Precision（精确率）**与 **Recall（召回率）**是另外两项基础指标。精确率反映模型输出的检测结果中有多少是真正正确的，召回率则衡量所有真实存在的目标中有多少被模型成功检测到。两者之间通常存在权衡关系：如果模型过于保守，精确率可能较高，但召回率会下降；如果模型倾向于输出更多候选目标，则召回率可能提高，但误检也会相应增加。为了综合描述这一关系，通常使用 **PR 曲线（Precision-Recall Curve）** 进行刻画，并进一步计算 **AP（Average Precision）** 和 **mAP（mean Average Precision）** 作为目标检测的标准综合评价指标。

由于检测器通常会输出大量候选框，其中许多框往往指向同一个目标，因此还需要使用后处理算法去除冗余结果。最经典的方法是 **NMS（Non-Maximum Suppression，非极大值抑制）**。其基本思想是：对于一组高度重叠的候选框，只保留置信度最高的框，将其余重叠度过高的候选框剔除。NMS 在传统检测器和多数深度检测器中都发挥着关键作用，它能够有效缓解“一个目标被重复检测多次”的问题。

另一个与候选框筛选相关的重要概念是 **objectness**。所谓 objectness，可以理解为“一个区域是否具有目标属性”的程度，它不直接区分类别，而是首先判断某一区域是否可能包含某个物体。引入 objectness 的意义在于先排除大量明显不可能含有目标的区域，从而显著降低后续分类与边界框回归的计算负担。这一思想在候选区域生成与现代检测器中的区域提议机制中都具有重要意义。

## 三、传统目标检测方法：滑动窗口与 HoG 特征

在深度学习方法兴起之前，目标检测主要依赖“**滑动窗口 + 人工特征 + 分类器**”这一技术路线。其基本流程是：首先在图像上使用固定大小的窗口按不同位置和尺度进行遍历；随后对每个窗口提取手工设计的特征表示；最后利用分类器判断当前窗口中是否存在目标。这种方法概念直观，但由于需要对图像中大量位置和尺度进行穷举搜索，因此计算成本较高。

在传统方法中，**HoG（Histogram of Oriented Gradients，方向梯度直方图）** 是最具代表性的特征描述子之一。HoG 的核心思想在于：目标的局部形状信息可以通过图像梯度方向的统计分布来表征。与直接使用原始像素值不同，HoG 更关注图像中亮度变化形成的边缘与轮廓，因此对于形状结构明显的目标，尤其是人体和行人检测任务，具有较强的表达能力。

HoG 的计算过程可分为以下几个步骤。首先，对图像中每个像素计算梯度，包括梯度幅值与梯度方向。梯度计算的本质是利用局部差分算子估计亮度变化，从而提取边缘信息。其次，将图像划分为若干个较小的局部区域，即 **cell**。在每个 cell 内，根据像素的梯度方向构建方向直方图，并以梯度幅值作为投票权重，从而统计该局部区域内的边缘方向分布。随后，将若干相邻的 cell 组合为更大的 **block**，对 block 内的特征进行归一化处理，以减弱光照变化、阴影和对比度差异带来的影响。最后，将整张检测窗口中的多个 block 特征拼接起来，形成高维特征向量，并送入分类器（如线性 SVM）进行目标判断。

HoG 的重要意义在于，它成功证明了局部梯度方向分布是刻画目标形状结构的一种有效表示。然而，HoG 也存在明显局限：其特征表达能力依赖人工设计，难以适应复杂背景、遮挡、形变和多类别目标场景；同时，滑动窗口式搜索也带来了较高计算成本。因此，随着深度学习的发展，目标检测逐渐从人工特征方法过渡到基于神经网络的特征自动学习方法。

## 四、深度学习时代的两阶段检测器：R-CNN 系列

目标检测进入深度学习阶段的重要标志之一是 **R-CNN（Regions with CNN features）** 的提出。其核心思想是，不再在整张图像上进行密集滑动窗口搜索，而是首先生成一批可能包含目标的候选区域（region proposals），然后对这些候选区域分别利用卷积神经网络提取特征，并完成类别判断与边界框精细回归。这种方法在本质上将检测任务分解为两个阶段：首先回答“哪些区域可能包含目标”，然后进一步回答“这些区域中是什么目标，以及目标边界应该如何修正”。

基于这一思想发展起来的方法通常被称为**两阶段检测器（two-stage detectors）**。其中，第一阶段负责候选区域生成，第二阶段负责分类与定位精修。两阶段方法通常具有较高检测精度，因为它们可以先粗筛候选区域，再对较少数量的高质量候选进行更细致的判断。

R-CNN 的后续发展包括 **Fast R-CNN** 和 **Faster R-CNN**。Fast R-CNN 通过在整张图像上共享卷积特征图，避免对每个候选区域单独运行 CNN，从而显著降低了计算开销。Faster R-CNN 则进一步提出 **RPN（Region Proposal Network）**，使用神经网络本身生成候选区域，取代了传统外部 proposal 算法，使得候选区域生成过程也被纳入统一的网络框架之中。这样，检测系统的端到端程度进一步提高，整体效率和准确率都得到了明显改进。

在这类方法中，**anchor box（锚框）** 是一个关键概念。锚框可以看作在特征图不同位置预设的一组参考边界框，具有不同尺度和长宽比。模型并非直接从零开始预测目标框，而是学习如何在这些参考框基础上进行位置偏移与尺度调整。锚框机制在很长一段时间内是主流检测器的重要组成部分，它提高了边界框回归的稳定性，但也带来了额外的人工设计成本，例如锚框数量、尺度分布和宽高比的选择等问题。

总体而言，两阶段检测器代表了目标检测精度优先的发展方向，其优势在于定位准确、复杂场景适应性较强，尤其适用于对检测性能要求较高的任务；其不足则主要体现在结构较复杂、推理速度相对较慢。

## 五、单阶段检测器：YOLO 与 SSD

与两阶段检测器不同，**单阶段检测器（one-stage detectors）** 的目标是直接在单次前向传播中完成目标分类与定位，不再显式依赖候选区域生成步骤。其中，最具代表性的模型是 **YOLO（You Only Look Once）**。

YOLO 的核心思想是将目标检测整体视为一个直接回归问题：模型输入整幅图像后，直接输出若干候选框的位置、类别以及相应置信度。由于省去了 proposal 生成与候选区域重复处理过程，YOLO 在速度方面具有显著优势，尤其适合实时检测场景，例如视频监控、机器人感知和自动驾驶等应用。其名称“只看一次”也正反映了该模型试图以最直接方式完成检测任务的设计理念。

YOLO 的高效性来自于整个检测流程的统一化。模型通常将图像划分为若干网格单元，每个单元负责预测一定数量的候选边界框及其类别概率。与两阶段方法相比，这种结构大幅减少了重复计算，因此可以达到较高的帧率。然而，早期 YOLO 虽然速度优势明显，但在检测精度方面，尤其是小目标检测、密集目标场景和定位精细度方面，通常不及两阶段检测器。这说明高效率和高精度之间往往存在一定权衡。

从训练角度看，YOLO 的损失函数通常由多个部分组成，包括类别预测误差、边界框位置误差、边界框尺寸误差以及目标存在与否的置信度误差。这一损失结构再次体现了目标检测本质上是分类与回归联合优化的问题。

在单阶段检测器的发展过程中，**SSD（Single Shot MultiBox Detector）** 是另一项重要工作。SSD 延续了单阶段、端到端的检测思路，但在结构设计上对多尺度特征进行了更有效利用。其核心思想是在不同分辨率的特征图上同时进行预测：较浅层特征图保留更高空间分辨率，更适合检测小目标；较深层特征图具有更强语义信息，更适合检测大目标。通过这种多尺度特征融合，SSD 在保持较高检测速度的同时，有效改进了对不同尺寸目标的检测能力。

SSD 也采用了类似默认框（default boxes）的机制，本质上仍可视为对锚框思想的继承与扩展。因此，可以认为 SSD 是单阶段检测器在速度与精度平衡方面的一次重要推进，它弥补了早期 YOLO 在精度方面的部分不足。

## 六、Transformer 在目标检测中的引入：DETR

近年来，Transformer 架构在自然语言处理领域取得巨大成功后，逐渐被引入视觉任务。其核心机制 **self-attention** 使模型能够在全局范围内建模不同位置之间的依赖关系，而这对于目标检测同样具有重要意义。图像中的目标往往并非孤立存在，不同区域之间的语义关系、上下文信息以及空间关联都可能影响检测结果。因此，Transformer 提供了一种不同于传统卷积局部感受野机制的全局建模方式。

在目标检测领域，最具代表性的 Transformer 模型是 **DETR（Detection Transformer）**。DETR 的核心贡献在于将目标检测重新表述为一个**集合预测（set prediction）**问题。传统检测器往往依赖锚框设计和 NMS 等后处理机制，而 DETR 则试图通过统一的端到端框架，直接输出一组无序的检测结果，并利用集合匹配策略完成训练监督。

DETR 的整体流程通常包括以下几个部分。首先，使用卷积神经网络作为 backbone 提取图像特征图。然后，将特征图展平并加入位置编码，以便 Transformer 能够保留空间位置信息。接着，这些序列化后的特征输入 Transformer encoder，用于建模图像全局上下文关系。随后，模型引入一组可学习的 **object queries**，将其送入 decoder，与编码后的图像特征进行交互。每个 query 最终输出一个检测结果，包括类别预测和边界框参数。

在该结构中，decoder 中的 **self-attention** 允许不同 object queries 之间相互作用，从而减少多个 query 反复关注同一目标的情况；**cross-attention** 则使每个 query 能够从图像特征中聚焦与自己最相关的区域。通过这种机制，DETR 以一种更加统一的方式实现了候选目标建模与最终检测输出。

DETR 的重要意义在于，它显著削弱了传统检测器中大量手工设计组件的作用，例如 anchor box、proposal generation 和 NMS，从而推动目标检测向更纯粹的端到端学习框架演进。然而，DETR 早期也存在训练收敛较慢、对小目标不够敏感等问题，这也促使后续研究继续围绕 Transformer 检测器进行优化。

## 七、DINO 及 Transformer 检测器的进一步发展

为克服 DETR 训练效率与性能上的局限，研究者提出了一系列改进模型，其中 **DINO（DETR with Improved DeNoising Anchor Boxes）** 是具有代表性的工作之一。DINO 延续了 query-based Transformer detection 的基本思路，但在训练机制、query 设计以及边界框学习策略等方面进行了更系统的改进。

DINO 的关键思想之一是通过更有效的去噪训练（denoising training）提高模型的学习效率。与早期 DETR 相比，DINO 更重视 query 初始化与目标匹配过程的稳定性，使模型能够更快收敛到高质量检测结果。可以认为，DINO 的出现标志着 Transformer 检测框架从“概念上可行”逐步走向“性能上具备竞争力”。

讲座中还提到了 **DS-Det** 等较新的研究工作。这类方法表明，当前研究前沿已不再局限于是否使用 Transformer，而是进一步关注：在 query-based 检测范式中，查询向量应如何定义，注意力机制如何解耦，不同查询之间的功能分工如何设计，以及如何在复杂场景下提升检测灵活性与泛化能力。这说明 Transformer 检测器的发展已经进入更细粒度的结构设计与机制优化阶段。

## 八、从封闭集检测到开放词汇检测

传统目标检测大多属于 **closed-set detection（封闭集检测）**。在这种设定下，训练阶段的类别集合是固定的，测试时默认也只会遇到这些已知类别。然而，真实环境远比标准数据集复杂，系统往往会遇到训练时未出现过的新目标类别。因此，如何让检测器具备识别未知类别或由语言描述定义的新类别的能力，成为近年来的重要研究方向。

在这一背景下，**open-set detection** 与 **open-vocabulary detection（开放词汇检测）** 逐渐兴起。其核心思想是通过引入**文本编码器（text encoder）**，将类别名称、属性描述甚至自然语言短语映射到语义向量空间，再与图像特征进行对齐。这样，模型不再严格依赖固定类别标签进行识别，而是能够利用语言语义实现对新类别的泛化。

这一趋势意味着目标检测正在从“封闭类别识别”迈向“开放世界理解”。检测器的能力边界不再由预定义类别表严格限定，而是可以通过语言描述进行扩展，这对于真实场景部署具有重要意义。

## 九、Grounding DINO 与视觉语言融合检测

在开放词汇检测的发展过程中，**Grounding DINO** 是一个极具代表性的模型。它将图像特征提取、目标检测框架与语言条件建模相结合，使模型能够根据文本描述在图像中定位对应目标。与传统检测器仅输出固定类别标签不同，Grounding DINO 能够根据用户给定的语言提示，完成更灵活的目标定位任务。

Grounding DINO 的重要性体现在两个方面。首先，它体现了检测任务从“类别识别”向“语言条件下的视觉定位”转变。模型不仅需要识别目标，还需要理解文本描述中的属性、关系与语义限定。其次，它展示了视觉与语言深度融合的趋势。文本特征不再只是辅助标签，而是真正参与到目标查询、特征交互和检测决策过程中。

这一类方法使得模型可以根据自然语言描述定位特定对象，例如具有某种颜色、位于某个位置、与其他物体存在某种关系的目标。由此可见，目标检测正在从传统的封闭类别识别问题发展为更一般的跨模态视觉理解问题。

## 十、未来发展方向：开放环境、关系建模与生成式融合

从讲座内容看，目标检测的未来发展方向大致可以概括为三个方面。

第一，是**面向开放环境的泛化能力提升**。在真实场景中，模型需要适应新类别、新背景、新任务与新环境，这要求检测器具备更强的跨域泛化与开放集识别能力。特别是在具身智能与机器人应用中，系统往往需要在未充分标注的新场景中工作，因此“训练于有限数据，应用于开放环境”将成为持续存在的核心挑战。

第二，是**关系建模与空间感知能力增强**。单纯识别物体类别和位置已经不足以满足复杂任务需求，未来的视觉系统还需要理解目标之间的语义关系、空间关系以及上下文约束。例如，“桌子上的杯子”“站在车旁的人”“靠近门的箱子”等描述都体现出检测任务与关系理解的融合趋势。

第三，是**与生成模型的结合**。随着扩散模型等生成式方法的发展，视觉理解与视觉生成之间的界限正在逐渐模糊。未来目标检测可能与生成式先验、跨模态语义生成、区域生成式建模等方向深度结合，从而在少样本检测、开放词汇理解和复杂场景推理中获得更强能力。

## 十一、总结

总体而言，本次讲座以“目标检测的前世与今生”为主线，较为完整地呈现了该领域的发展历程与技术逻辑。传统阶段的代表方法是基于滑动窗口和人工特征的 HoG 检测框架，它奠定了局部梯度结构在视觉检测中的基础地位。深度学习阶段则首先通过 R-CNN 系列确立了两阶段检测器的高精度范式，随后又通过 YOLO 和 SSD 推动检测系统向实时化、高效率方向发展。进一步地，Transformer 的引入使检测从复杂的多模块流水线逐渐走向统一的端到端建模，而 DINO 等工作则持续优化这一范式的性能与训练效率。最后，Grounding DINO 和开放词汇检测的发展表明，目标检测正从固定类别识别迈向语言视觉融合、开放世界感知和更高级语义理解。

因此，目标检测的发展并不仅仅体现为模型结构的更新换代，更体现为视觉系统能力边界的不断扩展：从局部边缘到深层特征，从封闭类别到开放语义，从静态图像分析到面向真实环境的智能感知。对于计算机视觉研究而言，这一演化过程具有非常重要的方法论意义，也为未来多模态感知与具身智能系统的发展提供了坚实基础。
